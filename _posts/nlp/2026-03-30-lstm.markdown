## ☺️ 들어가며
지난 포스팅에서 우리는 RNN 이 왜 과거의 정보를 쉽게 잊어버리는지, 그리고 그 수학적 원인인 기울기 소실에 대해 깊이 있게 다루었습니다.

긴 시퀀스를 처리할 때 발생하는 이 고적인 문제를 해결하기 위해 등장한 것이 바로 **LSTM (Long Short-Term Memory)** 입니다. 오늘은 LSTM이 어떻게 **Gate** 라는 혁신적인 구조를 통해 정보의 선별적 기억과 망각을 제어하는지, 그리고 **Cell State** 를 통해 어떻게 기울기를 멀리까지 전달하는지 자세히 알아보겠습니다.

---

## ☘️ LSTM의 개념
LSTM (Long Short-Term Memory) 은 시계열 데이터나 자연어와 같은 시퀀스 데이터를 처리할 때, 정보의 흐름을 정교하게 제어하기 위해 설계된 구조입니다. 핵심은 단순히 모든 정보를 받아들이는 것이 아니라, '무엇을 기억하고 무엇을 버릴지' 스스로 결정한다는 점에 있습니다.

이를 위해 LSTM은 정보를 아주 오랫동안 변함없이 전달할 수 있는 전용 통로인 셀 상태 (Cell State) 와, 이 통로의 정보를 관리하는 게이트 (Gate) 시스템을 도입했습니다. 결과적으로 이름 그대로 단기 기억 (Short-term Memory) 을 장기적으로 (Long) 유지할 수 있는 메커니즘을 갖추게 되었습니다.

---

## 🏭 셀 상태 (Cell State)
LSTM의 전체 구조를 관통하는 가장 중요한 개념은 바로 셀 상태 (Cell State) 입니다.

### 🟩 정보의 컨베이어 벨트
셀 상태는 LSTM 모듈의 최상단을 관통하는 수평선으로 시각화됩니다. 이 선은 시퀀스의 시작부터 끝까지 정보를 전달하는 **장기 기억(Long-term Memory)** 전용 통로입니다.

* **직관적 이해:** RNN이 매번 새로운 정보를 이전 정보 위에 덧칠하여 과거가 흐릿해지는 방식이라면, LSTM은 과거 정보를 담은 컨베이어 벨트를 옆에 두고, 필요한 위치에서만 아주 미세하게 정보를 수정(더하거나 빼기)합니다.

* **선형적 흐름:** 이 벨트 위에서 정보는 비선형 활성화 함수 (Sigmoid나 Tanh) 를 거치지 않고 선형적으로 흐릅니다. 덕분에 정보의 왜곡이나 손실이 매우 적습니다.

### 🟩 기울기 소실 문제의 기하학적 해결
RNN의 기울기 소실 원인은 가중치 $W_h$ 의 반복적인 곱셈에 있었습니다. LSTM은 이 문제를 덧셈으로 해결했습니다.

뒤에서 수식으로 확인하겠지만, LSTM의 셀 상태는 역전파 과정에서 기울기가 전달될 때, 복잡한 행렬 곱셈 대신 **덧셈 연산** 을 기반으로 정보를 전달합니다. 덕분에 기울기가 소멸되지 않고 시퀀스의 앞부분까지 효과적으로 도달할 수 있게 됩니다. 이것이 바로 LSTM이 장기 의존성 문제를 해결한 방법입니다.

---

## 🚪 게이트 (Gates)
이제 Cell State 라는 컨베이어 벨트 위에서 정보를 조절하는 게이트 (Gate) 메커니즘에 대해 알아보겠습니다. LSTM에는 총 3개의 게이트가 존재하며, 각각 망각, 저장, 출력의 역할을 담당합니다. 모든 게이트는 시그모이드 (Sigmoid) 활성화 함수를 사용하여 $0$ (차단)에서 $1$ (통과) 사이의 값을 출력함으로써 정보의 양을 조절합니다.

### 🟩 게이트의 종류 

#### ◼️ 망각 게이트 (Forget Gate, $f^{(t)}$)

$$f^{(t)} = \sigma (W_f h^{(t-1)} + U_f x^{(t)} + b_f)$$

가장 먼저 만나는 게이트로, 이전 시점의 정보 중 **무엇을 버릴지** 결정합니다. 시그모이드 출력값이 $0$ 에 가까울수록 이전 정보를 많이 삭제하고, $1$ 에 가까울수록 잘 유지합니다.

#### ◼️ 입력 게이트 (Input Gate, $i^{(t)}$) 

$$i^{(t)} = \sigma (W_i h^{(t-1)} + U_i x^{(t)} + b_i)$$

현재 시점의 새로운 정보 중 **무엇을 Cell State에 저장** 할지 결정합니다. 어떤 값을 업데이트할지 선택하는 필터 역할을 수행합니다.

#### ◼️ 출력 게이트 (Output Gate, $o^{(t)}$)

$$o^{(t)} = \sigma (W_o h^{(t-1)} + U_o x^{(t)} + b_o)$$

업데이트된 Cell State를 바탕으로, 최종적으로 외부 (다음 시점의 은닉 상태) 로 내보낼 값 을 결정합니다. 전체 기억 중 현재 예측에 당장 필요한 정보만 걸러내어 출력합니다.

### 🟩 게이트 학습의 최종 원리
위 수식들을 보면 알 수 있듯이, 모든 게이트는 동일한 입력($h^{(t-1)}, x^{(t)}$)을 받습니다. 하지만 모델이 학습됨에 따라 각 게이트는 서로 다른 가중치($W, U$)와 편향($b$) 값을 갖게 됩니다.

*  **서로 다른 판단:** 동일한 상황에서도 망각 게이트는 "과거 정보를 버려라"고 판단하고, 입력 게이트는 "이 정보는 중요하니 저장해라"고 판단할 수 있는 이유가 바로 이 독립적인 가중치 덕분입니다.

* **학습의 목표:** 결과적으로 LSTM의 학습이란, 어떤 상황에서 정보를 버리고(Forget), 저장하고(Input), 내보낼지(Output) 결정하는 이 최적의 가중치 조합을 찾아가는 과정이라고 할 수 있습니다.

## ⚙️ 상태 업데이트 과정
게이트의 판단이 끝났다면, 이제 실제로 Cell State ($C^{(t)}$) 와 Hidden State ($h^{(t)}$) 를 업데이트할 차례입니다. 이 과정은 크게 과거 정보를 얼마나 유지할지, 새로운 정보를 얼마나 더할지, 그리고 최종적으로 무엇을 출력할지 의 3단계로 요약됩니다.

### 🟩 Cell State 업데이트
가장 먼저, 이전 시점의 Cell State ($C^{(t-1)}$) 를 새로운 Cell State ($C^{(t)}$) 로 갱신합니다. 이 과정은 망각 게이트와 입력 게이트의 결과값을 활용합니다.

#### ◼️ 1. 과거 기억 정제
이전 Cell State 에 망각 게이트 값($f^{(t)}$) 을 곱하여 버릴 정보를 제거합니다.

#### ◼️ 2. 새로운 후보 생성
현재 입력과 이전 은닉 상태를 조합하여 새로운 기억 후보($\tilde{C}^{(t)}$) 를 만듭니다.

$$\tilde{C}^{(t)} = \tanh(W_c h^{(t-1)} + U_c x^{(t)} + b_c)$$

#### ◼️ 3. 기억 병합
정제된 과거 기억에 입력 게이트($i^{(t)}$) 를 통해 선별된 새로운 후보 값을 더해 최종적인 $C^{(t)}$ 를 완성합니다.

$$C^{(t)} = f^{(t)} \odot C^{(t-1)} + i^{(t)} \odot \tilde{C}^{(t)}$$

### 🟩 Hidden State 업데이트
마지막으로, 업데이트된 Cell State 를 바탕으로 현재 시점의 출력값인 Hidden State ($h^{(t)}$) 를 계산합니다.

#### ◼️ 1. 비선형 변환
Cell State ($C^{(t)}$) 를 $\tanh$ 에 통과시켜 값의 범위를 $-1$ 에서 $1$ 사이로 조정합니다.

#### ◼️ 2. 최종 필터링
앞서 계산한 출력 게이트($o^{(t)}$) 값을 곱하여, 전체 기억 중 외부로 내보낼 정보만 걸러냅니다.

$$h^{(t)} = o^{(t)} \odot \tanh(C^{(t)})$$

## 🌟 LSTM이 기울기 소실을 해결한 법
왜 LSTM 은 수백 단어 뒤에서도 문장의 첫 주어를 잊지 않을까요? 그 비밀은 셀 상태 (Cell State) 의 선형적인 정보 흐름과 덧셈 기반의 업데이트에 있습니다.

### 🟩 지워지지 않는 정보의 통로
만약 특정 차원의 망각 게이트($f^{(t)}$) 가 $1$ 로 유지되고 입력 게이트($i^{(t)}$) 가 $0$ 이라면, 그 정보는 아무런 변형이나 감쇄 없이 다음 시점으로 무한히 전달됩니다. 이는 필요한 정보를 창고에 보관했다가 수십 단계 뒤에 꺼내 쓰는 것과 같습니다.

### 🟩 뎃셈의 미학
지난번에 다룬 Vanilla RNN 의 치명적인 약점은 역전파 시 기울기가 곱셈 연쇄를 타며 기하급수적으로 소멸된다는 것이었습니다. 반면 LSTM 은 다음과 같은 구조적 이점을 가집니다.

* **기울기 방어:** 셀 상태를 업데이트할 때 덧셈 연산을 축으로 삼기 때문에, 역전파 시 기울기가 곱셈에 의해 파묻히지 않고 과거로 원활하게 흘러갈 수 있습니다.

* **지름길 확보:** 게이트들이 열려 있는 한, 에러 신호는 마치 고속도로를 달리는 것처럼 시퀀스의 앞부분까지 에러를 전파합니다.

### 🟩 실질적 성능의 차이
실제로 Vanilla RNN 이 고작 7개 단어 내외의 의존성만 파악하는 데 반해, LSTM 은 약 100 시점 이상의 긴 맥락도 거뜬히 소화해 냅니다.

## 😙 마치며
오늘은 RNN 의 한계를 극복하고 자연어 처리의 패러다임을 바꿨던 LSTM (Long Short-Term Memory) 에 대해 자세히 알아보았습니다.

LSTM 이 셀 상태 (Cell State) 와 게이트 (Gate) 라는 혁신적인 구조를 도입함으로써, 단순히 정보를 수동적으로 받아들이는 수준을 넘어 '무엇을 잊고 무엇을 기억할지' 모델 스스로 판단하게 되었다는 점을 확인할 수 있었습니다. 이러한 선별적 기억 메커니즘 덕분에 수백 단계 이전의 정보도 잃어버리지 않고 학습에 반영할 수 있게 된 것이죠.

하지만 현실의 데이터는 앞뒤 문맥을 동시에 파악해야 하거나, 더 복잡한 추론을 필요로 하기도 합니다. 그래서 다음 포스팅에서는 시퀀스의 정보를 양방향에서 읽어 들이는 Bidirectional RNN 과, 층을 깊게 쌓아 고차원적인 특징을 추출하는 Multi-layer RNN (Deep RNN) 에 대해 다루어 보겠습니다. 감사합니다.
