안녕하세요. 지난 포스팅에서는 RNN의 기본 구조와 데이터를 순차적으로 처리하는 원리에 대하여 알아보았습니다. 

하지만, RNN을 실제로 학습시키다 보면, 문장이 길어질수록 앞부분의 정보를 뒤로 전달하지 못하는 한계에 부딪히게 됩니다. 오늘은 RNN이 가중치를 업데이트하는 핵심 메커니즘인 **BPTT (Backpropagation Through Time)** 를 수학적으로 뜯어보고, 이 과정에서 발생하는 고질적인 문제인 **기울기 소실 (Vanishing Gradient)** 과 **기울기 폭발 (Gradient Exploding)** 현상을 심도 있게 다루어 보겠습니다.

## 🅱️ BPTT (Backpropagtion Through Time)

### 🟩 BPTT의 개념
RNN 은 각 타임스텝마다 입력 $x_t$ 와 이전 시점의 은닉 상태 $h_{t-1}$ 를 받아 현재의 은닉 상태 $h_t$ 를 갱신하는 구조를 가집니다. 이때 핵심은 모든 타임스텝에서 동일한 가중치 행렬($W_h, W_x, W_y$ 등)을 공유한다는 점입니다.

따라서 모델을 학습시키기 위한 역전파 과정에서도 일반적인 신경망과는 다른 접근이 필요합니다. 마지막 시점의 손실 (Loss) 로 부터 시작하여, 은닉 상태가 갱신되어 온 역방향 경로를 따라 과거의 모든 타임스텝으로 오차를 전파하며 가중치의 기울기를 계산해야합니다. 이처럼 시간을 거슬러 올라가며 수행되는 역전파 알고리즘을 **BPTT (Backpropagation Through Time)** 라고 부릅니다.

### 🟩 BPTT의 수학적 정리
전체 손실 함수 $E$ 에 대한 가중치 $W$ 의 기울기를 구하기 위해서는 각 시점 $t$ 에서 발생하는 손실 $E_t$ 에 대한 미분값을 합산해야 합니다.

$$\frac{\partial E}{\partial W} = \sum_{t=1}^{T} \frac{\partial E_t}{\partial W}$$

연쇄 법칙 (Chain Rule)을 적용하여 특정 시점 $t$ 에서의 기울기를 분석해 보면 다음과 같은 전개식을 얻을 수 있습니다.

$$\frac{\partial E_t}{\partial W} = \sum_{k=1}^{t} \frac{\partial E_t}{\partial y_t} \frac{\partial y_t}{\partial h_t} \left( \prod_{i=k+1}^{t} \frac{\partial h_i}{\partial h_{i-1}} \right) \frac{\partial h_k}{\partial W}$$

* **$\frac{\partial E_t}{\partial y_t} \frac{\partial y_t}{\partial h_t}$:** 현재 시점 $t$ 에서 출력과 은닉 상태 사이의 기울기입니다.

* $\prod_{i=k+1}^{t} \frac{\partial h_i}{\partial h_{i-1}}$: 시점 $t$ 에서 과거 시점 $k$ 까지 오차가 전달되기 위해 거쳐가는 은닉 상태 간의 변화량입니다. 이 곱연산($\prod$) 과정에서 동일한 가중치가 반복적으로 곱해지며 기울기 소실/폭발 문제가 발생하게 됩니다.

* **$\frac{\partial h_k}{\partial W}$:** 각 과거 시점 $k$ 에서 가중치가 은닉 상태에 미치는 직접적인 영향력입니다.

### 🟩 BPTT 적용 예시 ($t=3$ 일 때)
이해를 돕기 위해 세 번째 타임스텝 ($t=3$) 에서 발생하는 손실 $E_3$ 에 대해 가중치 $W_h$ 의 기울기를 구하는 과정을 단계별로 살펴보겠습니다.

$E_3$ 는 $h_3$ 에 의존하고, $h_3$ 는 다시 $h_2$ 에, $h_2$ 는 $h_1$ 에 의존하는 연속적인 구조를 가집니다. 따라서 $W_h$ 에 대한 미분은 각 시점까지 거슬러 올라가는 경로의 합으로 표현됩니다.

$$\frac{\partial E_3}{\partial W_h} = \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial W_h} + \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial h_2} \frac{\partial h_2}{\partial W_h} + \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial h_2} \frac{\partial h_2}{\partial h_1} \frac{\partial h_1}{\partial W_h}$$

* **첫 번째 항:** $h_3$ 단계에서 직접적으로 $W_h$ 가 기여한 정도를 계산합니다.
* **두 번째 항:** $h_2$ 단계에서 $W_h$ 가 기여한 오차가 $h_3$ 를 거쳐 전달된 양을 계산합니다.
* **세 번째 항:** $h_1$ 단계에서 $W_h$ 가 기여한 오차가 $h_2, h_3$ 를 모두 거쳐 전달된 양을 계산합니다.

이처럼 $t$가 커질수록 더 많은 과거의 항들이 더해지며, 각 항 내부의 미분 연쇄 ($\frac{\partial h_i}{\partial h_{i-1}}$)는 길어지게 됩니다. 이것이 바로 RNN 이 긴 시퀀스를 학습할 때 어려움을 겪는 근본적인 원인이 됩니다.

## 📉 기울기 소실 (Vanishing Gradient)

### 🟩 기울기 소실의 개념
기울기 소실이란 역전파 과정에서 정보가 전달될수록 기울기 값이 기하급수적으로 작아져, 결국 0에 수렴하게 되는 현상입니다. 이 문제가 발생하면 시퀀스 앞부분의 가중치들이 거의 업데이트되지 않아, 모델이 **장기 의존성 (Long-term Dependecny)를 학습하지 못하게 됩니다. 즉, 문장의 끝에서 발생한 에러가 문장 앞머리의 단어까지 도달하지 못하고 사라지는 것입니다.

### 🟩 수학적 증명
왜 이런 현상이 발생하는지, 계산을 단순화하기 위해 활성화 함수 $\sigma$ 가 선형 함수($\sigma(x) = x$)라고 가정하고 수학적으로 분석해 보겠습니다.

#### ◼️ 1. 은닉 상태 간의 관계식
RNN의 은닉 상태 식 $h^{(t)} = \sigma(W_h h^{(t-1)} + W_x x^{(t)} + b)$ 에서 $\sigma$ 가 항등 함수라면, 이전 시점에 대한 현재 시점의 미분값은 다음과 같이 가중치 행렬 $W_h$ 만 남게 됩니다.

$$\frac{\partial h^{(t)}}{\partial h^{(t-1)}} = diag(\sigma'(W_h h^{(t-1)} + W_x x^{(t)} + b)) W_h$$

여기서 $\sigma(x) = x$ 이면 $\sigma'(x) = 1$ 이므로, 결과적으로 $\frac{\partial h^{(t)}}{\partial h^{(t-1)}} = W_h$ 가 됩니다.

#### ◼️ 2. 체인 룰을 통한 기울기 전개
시점 $i$ 에서 발생한 손실 $J^{(i)}$ 에 대해, $l = i - j$ 만큼 떨어진 과거 시점 $j$ 의 은닉 상태 $h^{(j)}$ 가 미치는 영향($\frac{\partial J^{(i)}}{\partial h^{(j)}}$)을 구해보면 다음과 같습니다.

$$\frac{\partial J^{(i)}}{\partial h^{(j)}} = \frac{\partial J^{(i)}}{\partial h^{(i)}} \prod_{j < t \le i} \frac{\partial h^{(t)}}{\partial h^{(t-1)}} = \frac{\partial J^{(i)}}{\partial h^{(i)}} W_h^{i-j} = \frac{\partial J^{(i)}}{\partial h^{(i)}} W_h^{l}$$

여기서 핵심은 가중치 행렬 $W_h$ 가 $l$ 번 거듭제곱된다는 점입니다.

#### ◼️ 3. 고유값 (Eigenvalue) 분석: 소실의 결정적 원인
가중치 행렬 $W_h$ 의 고윳값을 $\lambda_1, \lambda_2, \dots, \lambda_n$ 이라고 할 때, 이 기울기 값은 고윳값의 거듭제곱에 비례하게 됩니다.

* **만약 모든 고윳값이 1보다 작다면 ($\lambda_i < 1$):** $l$ 이 커짐(과거로 멀리 갈수록)에 따라 $W_h^l$ 은 기하급수적으로 작아져 결국 0에 수렴합니다.

* **실제 상황:** 비선형 활성화 함수(Sigmoid, Tanh)를 사용할 때도 원리는 같습니다. $\sigma'$ 의 최댓값이 1 이하(Sigmoid는 0.25, Tanh는 1)이기 때문에, $W_h$ 의 값이 작을수록 기울기는 훨씬 더 빠르게 사라집니다.

### 🟩 요약 및 시사
결국 RNN에서 기울기 소실이 발생하는 근본적인 이유는 동일한 가중치 행렬 $W_h$ 가 매 타임스텝마다 반복해서 곱해지는 구조 때문입니다.

* **수학적 결과:** $\frac{\partial J^{(i)}}{\partial h^{(j)}} \approx 0$ (for large $l$)

* **상황 예시:** "The cat, which ate a lot of food, was full." 이라는 문장에서 'cat'과 'was' 사이의 거리가 멀 경우, 모델은 'was'를 결정할 때 'cat'의 정보를 거의 활용하지 못하게 됩니다.

## 🧨 기울기 폭발 (Gradient Exploding)

### 🟩 기울기 폭발의 정의
앞서 살펴본 기울기 소실과 반대로, 가중치 행렬 $W_h$ 의 고윳값이 1보다 큰 경우 기울기가 기하급수적으로 커지는 현상이 발생합니다. 이를 기울기 폭발이라고 합니다.

### 🟩 기울기 폭발이 문제인 이유E_3$ 는 $h_3$ 에 의존하고, $h_3$ 는 다시 $h_2$ 에, $h_2$ 는 $h_1$ 에 의존하는 연속적인 구조를 가집니다. 따라서 $W_h$ 에 대한 미분은 각 시점까지 거슬러 올라가는 경로의 합으로 표현됩니다.

$$\frac{\partial E_3}{\partial W_h} = \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial W_h} + \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial h_2} \frac{\partial h_2}{\partial W_h} + \frac{\partial E_3}{\partial h_3} \frac{\partial h_3}{\partial h_2} \frac{\partial h_2}{\partial h_1} \frac{\partial h_1}{\partial W_h}$$

* **첫 번째 항:** $h_3$ 단계에서 직접적으로 $W_h$ 가 기여한 정도를 계산합니다.
* **두 번째 항:** $h_2$ 단계에서 $W_h$ 가 기여한 오차가 $h_3$ 를 거쳐 전달된 양을 계산합니다.
* **세 번째 항:** $h_1$ 단계에서 $W_h$ 가 기여한 오차가 $h_2, h_3$ 를 모두 거쳐 전달된 양을 계산합니다.

이처럼 $t$가 커질수록 더 많은 과거의 항들이 더해지며, 각 항 내부의 미분 연쇄 ($\frac{\partial h_i}{\partial h_{i-1}}$)는 길어지게 됩니다. 이것이 바로 RNN 이 긴 시퀀스를 학습할 때 어려움을 겪는 근본적인 원인이 됩니다.

## 📉 기울기 소실 (Vanishing Gradient)

### 🟩 기울기 소실의 개념
기울기 소실이란 역전파 과정에서 정보가 전달될수록 기울기 값이 기하급수적으로 작아져, 결국 0에 수렴하게 되는 현상입니다. 이 문제가 발생하면 시퀀스 앞부분의 가중치들이 거의 업데이트되지 않아, 모델이 **장기 의존성 (Long-term Dependecny)를 학습하지 못하게 됩니다. 즉, 문장의 끝에서 발생한 에러가 문장 앞머리의 단어까지 도달하지 못하고 사라지는 것입니다.

### 🟩 수학적 증명
왜 이런 현상이 발생하는지, 계산을 단순화하기 위해 활성화 함수 $\sigma$ 가 선형 함수($\sigma(x) = x$)라고 가정하고 수학적으로 분석해 보겠습니다.

#### ◼️ 1. 은닉 상태 간의 관계식

여기서 $\sigma(x) = x$ 이면 $\sigma'(x) = 1$ 이므로, 결과적으로 $\frac{\partial h^{(t)}}{\partial h^{(t-1)}} = W_h$ 가 됩니다.

$$\frac{\partial J^{(i)}}{\partial h^{(j)}} = \frac{\partial J^{(i)}}{\partial h^{(i)}} \prod_{j < t \le i} \frac{\partial h^{(t)}}{\partial h^{(t-1)}} = \frac{\partial J^{(i)}}{\partial h^{(i)}} W_h^{i-j} = \frac{\partial J^{(i)}}{\partial h^{(i)}} W_h^{l}$$

결국 RNN에서 기울기 소실이 발생하는 근본적인 이유는 동일한 가중치 행렬 $W_h$ 가 매 타임스텝마다 반복해서 곱해지는 구조 때문입니다.

## 🧨 기울기 폭발 (Gradient Exploding)

### 🟩 기울기 폭발의 정의
앞서 살펴본 기울기 소실과 반대로, 가중치 행렬 $W_h$ 의 고윳값이 1보다 큰 경우 기울기가 기하급수적으로 커지는 현상이 발생합니다. 이를 기울기 폭발이라고 합니다.

### 🟩 기울기 폭발이 문제인 이유
기울기가 너무 커지면 경사 하강법 (SGD) 의 업데이트 단계에서 가중치가 비정상적으로 튀게 됩니다.

$$\theta^{new} = \theta^{old} - \alpha \nabla_{\theta}J(\theta)$$

* **Bad Updates:** 기울기($\nabla_{\theta}J(\theta)$ )가 과도하게 크면 업데이트 보폭($\alpha \nabla$)이 너무 넓어집니다. 이로 인해 최적의 파라미터 지점을 지나쳐버리고, 손실 (Loss) 이 급격히 커지는 엉뚱한 지점으로 이동하게 됩니다.

* **수치적 불안정성:** 최악의 경우 기울기 값이 컴퓨터가 표현할 수 있는 범위를 넘어서면서 네트워크 내에 Inf 또는 NaN (Not a Number) 값이 발생합니다. 이 단계에 이르면 학습을 중단하고 이전 체크포인트부터 다시 시작해야 하는 번거로움이 생깁니다.

### 🟩 Gradient Clipping
기울기 소실은 모델 구조를 바꿔야 하는 (LSTM 등) 어려운 문제인 반면, 기울기 폭발은 상대적으로 해결이 간단합니다. 바로 **Gradient Clipping** 기법을 사용하는 것입니다.

#### ◼️ 동작 원리
기울기의 노름 (Norm, $||\hat{g}||$) 이 특정 임계값 (threshold) 을 넘어서면, 기울기의 방향은 유지하되 크기를 강제로 줄여버리는 방식입니다.

$$if \ ||\hat{g}|| \ge threshold \ then$$

$$\hat{g} \leftarrow \frac{threshold}{||\hat{g}||} \hat{g}$$

* **방향 유지:** 기울기가 가리키는 방향은 여전히 손실을 줄이는 유효한 정보를 담고 있습니다.
* **보폭 조절:** 너무 큰 보폭으로 튀어 나가는 것만 방지하기 위해, 같은 방향으로 가되 더 작은 걸음을 내딛도록 강제합니다.

## ⁉️ 기울기 소실은 RNN 만의 문제인가?

### 🟩 기울기 소실은 모든 심층 신경망의 문제
기울기 소실은 계층 (Layer) 이 깊어지는 모든 심층 신경망 (Deep Neural Network) 이 직면하는 공통적인 과제입니다. 층이 깊어질수록 역전파되는 기울기가 앞쪽 레이어로 갈수록 희미해지는 현상은 어디에나 존재합니다.

하지만 RNN에서 이 문제가 유독 치명적인 이유는, 서로 다른 가중치를 사용하는 일반 DNN과 달리 동일한 가중치($W_h$)를 타임스텝마다 반복해서 곱하는 구조이기 때문입니다.

### 🟩 현대 신경망의 해결책: Skip Connection
현대 알고리즘들은 기울기 소실을 해결하기 위해 "기울기가 통과할 수 있는 통로"를 만드는 전략을 취합니다.

#### ◼️ Highway Networks
Gate 메커니즘을 도입하여, 특정 정보는 변형 없이 다음 레이어로 그대로 통과시킬지 결정합니다. 이는 나중에 LSTM의 Forget Gate 개념과도 맞닿아 있습니다.

#### ◼️ ResNet (Residual Networks)
Skip Connection을 도입하여 입력 $x$를 출력에 직접 더해주는($f(x) + x$) 구조를 가집니다. 미분 시 $+1$이라는 항이 살아남아 기울기가 아주 깊은 곳까지 소멸되지 않고 전달됩니다.

#### ◼️ DenseNet (Densly Connected Networks)
모든 레이어의 특징 맵을 이후의 모든 레이어에 결합(Concat)합니다. 앞쪽 레이어의 정보가 뒤쪽까지 직접적으로 연결되므로 기울기 소실이 획기적으로 줄어듭니다.
