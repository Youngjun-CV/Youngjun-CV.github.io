## 🫡 들어가며
지난 글에서 초기 언어 모델인 n-gram과 고정 창(Fixed-window) 기반 모델을 정리했었다. 이 모델들에게는 치명적인 단점이 있었는데, 입력받을 수 있는 시퀀스의 길이가 고정되어 있다는 것이었다. 즉, 모델이 정해놓은 길이를 벗어나는 긴 문장은 구조적으로 처리하기가 매우 까다로웠다.

이런 구조적인 한계를 극복하고, 문장의 길이에 관계없이 데이터를 처리하며 다음 단어를 예측할 수 있는 모델이 등장했다. 바로 **RNN (Recurrent Neural Network)** 이다. 이번 포스팅에서는 RNN이 어떤 구조를 가졌길래 길이의 제약을 극복할 수 있었는지 정리해 보도록 하겠다.

## 🌀 RNN (Recurrent Neural Network)

### 🟩 RNN의 등장 배경
위에서도 말했듯이 기존의 n-gram이나 고정 창 기반 모델은 미리 정해진 $n$ 개의 단어만 볼 수 있었다. 하지만 현실에서 사용하는 문장은 그 길이가 제각각이며, 때로는 아주 멀리 떨어진 단어가 현재 단어의 의미에 결정적인 영향을 주기도 한다. RNN은 바로 이 지점, **'가변적인 길이를 가진 시퀀스 데이터를 어떻게 효율적으로 처리할 것인가'** 라는 고민에서 등장했다.

### 🟩 핵심 아이디어: 가중치 공유 (Weight Sharing)
RNN의 구조를 자세히 들여다보기 전에 핵심 아이디어부터 짚고 넘어가자면, 바로 **모든 시점($t$)에서 동일한 가중치를 반복해서 사용한다** 는 것이다. 쉽게 말하면 같은 구조를 계속 반복해서 적용해 나간다는 말이다.

RNN은 매 단계마다 새로운 파라미터를 학습하는 대신, 하나의 파라미터를 모든 단계에서 공유한다. 이 방식이 혁신적인 이유는 두 가지다.

* **효율성:** 문장의 길이가 10이든 1,000이든 모델의 크기(파라미터 수)가 일정하게 유지된다.

* **일반화:** 특정 단어가 문장의 앞부분에 나오든 뒷부분에 나오든, 모델은 동일한 규칙으로 그 단어를 해석한다. 즉, 위치에 상관없이 언어의 보편적인 패턴을 학습할 수 있게 된다.

### 🟩 RNN의 구조
앞서 언급한 가중치 공유라는 특징 덕분에 RNN의 기본 구조는 매우 단순하다. 구조의 핵심은 모델이 현재의 입력값뿐만 아니라, **'과거의 기억'** 을 함께 고려한다는 것이다.

이 구조를 이해하기 위해 두 가지 관점으로 나누어 볼 수 있다.

* **재귀적 구조 (Recurrnece):** 모델이 자기 자신을 다시 참조하는 루프(Loop) 형태다. $t$ 시점의 출력이 다음 시점 $t+1$ 의 입력으로 다시 들어가는 모습이다.

* **펼쳐진 구조 (Unrolling):** 이 루프를 시간 순서대로 길게 펼쳐보면, 동일한 구조의 셀(Cell)들이 사슬처럼 연결된 모양이 된다. 여기서 중요한 건, 펼쳐진 모든 셀이 **동일한 가중치 ($W$)** 를 공유하고 있다는 점이다.

결국 RNN 은 "이전 단계에서 요약된 정보($h_{t-1}$)를 현재의 입력($x_t$)과 결합하여 새로운 요약 정보($h_t$)를 만드는 과정"을 무한히 반복하는 구조라고 할 수 있다.

### 🟩 단계별 수식 정의
이제 위에서 살펴본 RNN의 구조를 실제 계산 과정인 수식으로 파악해 보자. 크게 세 단계의 흐름으로 나뉜다.

#### ◼️ 단어 임베딩 (Word Embedding) 

$$e^{(t)} = E x^{(t)}$$

아무런 처리를 거치지 않은 단어 데이터는 그저 단어의 위치 정보만 가진 **원-핫 벡터($x^{(t)}$)** 상태다. 모델이 이 단어의 의미를 파악할 수 있도록, 임베딩 행렬($E$)과 곱해 차원이 축소된 **밀집 벡터(Dense Vector, $e^{(t)}$)** 로 변환해야 한다. 이 과정을 거치면 단어는 **학습 가능한 벡터** 형태가 된다.


#### ◼️ 은닉 상태 (Hidden State) 계산
 
$$h^{(t)} = \sigma(W_h h^{(t-1)} + W_e e^{(t)} + b_1)$$

앞서 RNN이 '과거의 기억'을 고려한다고 했는데, 그 기억이 저장되는 저장소가 바로 **은닉 상태($h^{(t)}$)** 다.

* 현재의 은닉 상태는 이전 시점의 기억($h^{(t-1)}$) 과 현재 들어온 입력($e^{(t)}$) 을 각각의 가중치와 곱하고 더해서 만들어진다.
* 이전 은닉 상태와 현재 새로 들어온 입력에 곱해지는 가중치 ($W_h, \ W_e$) 는, 각 입력들을 현재에 얼마나 반영할지 결정하는 파라미터이다.
* 여기에 비선형 활성화 함수($\sigma$)를 적용함으로써 복잡한 패턴을 학습할 수 있게 된다.

이 수식에서 $h$ 가 다시 $h$ 를 만드는 **재귀적 구조** 를 확인할 수 있고, 현재의 은닉 상태는 과거와 현재가 섞인 '압축된 메모리' 역할을 한다. 


#### ◼️ 출력 분포 (Output Distribution) 생성

$$\hat{y}^{(t)} = \text{softmax}(U h^{(t)} + b_2) \in \mathbb{R}^{|V|}$$

현재 시점의 은닉 상태($h^{(t)}$)를 계산했다면, 이제 이를 바탕으로 다음에 올 단어가 무엇일지 예측할 차례다.

* 은닉 상태에 가중치($U$)를 곱해 어휘 사전(Vocabulary) 크기만큼의 점수를 구한다.
* 이후 Softmax 함수를 통과시키면 각 단어가 나타날 확률 분포($\hat{y}^{(t)}$)가 계산된다.

여기서 확률이 가장 높은 단어가 다음 단어로 예측되며, 학습 시에는 실제 정답이, 추론 시에는 이 예측값이 다음 시점의 입력($x^{(t+1)}$)으로 들어가며 시퀀스를 이어가게 된다.

### 🟩 RNN의 주요 장점 ⁉️⁉️⁉️⁉️⁉️⁉️⁉️⁉️⁉️⁉️⁉️⁉️
RNN은 시퀀스 데이터 처리에 있어 다음과 같은 강력한 이점을 제공합니다.

* **입력 길이의 유연성:** 이론적으로 어떤 길이의 입력 문장이라도 처리할 수 있는 구조적 유연성을 가집니다.
* **장기 의존성 활용 가능성:** 은닉 상태를 통해 과거의 정보를 계속 전달하므로, 아주 먼 과거의 정보도 현재 예측에 활용할 수 있는 잠재력을 가집니다.
* **파라미터 효율성:** 입력이 아무리 길어져도 동일한 가중치 ($W_h, W_e, U$) 를 공유하므로 모델의 전체 파라미터 개수가 늘어나지 않습니다.

### 🟩 RNN의 한계점
하지만 실제 연구와 실무에서 RNN은 몇 가지 치명적인 문제에 직면하게 됩니다.

* **기울기 소실 문제 (Vanishing Gradient):** 문장이 길어질수록 과거의 정보가 여러 번의 연산을 거치며 희미해지거나 사라지는 현상이 발생합니다. 이로 인해 실제로는 먼 과거의 문맥을 완벽히 유지하기 어렵습니다.
* **병렬 처리 불가능:** $t$ 시점의 계산을 위해서는 반드시 $t-1$ 시점의 결과가 필요합니다. 이러한 순차적 특성 때문에 GPU 를 활용한 병렬 연산이 어려워 학습 속도가 매우 느립니다.
* **기울기 폭주 (Exploding Gradient):** 반대로 기울기가 너무 커져 모델의 가중치가 비정상적으로 커지는 문제가 발생하기도 합니다.

## ✏️ RNN 학습 하기

### 🟩 학습 과정
RNN 언어 모델의 학습은 연속적인 데이터를 처리하며 다음 단어를 예측하는 과정으로 진행됩니다.

#### 1. 코퍼스 입력 및 처리
모델은 전체 말뭉치 (Corpus) 를 입력받습니다. RNN은 순차적인 데이터를 처리하는 데 특화되어 있어, 문장의 처음부터 끝까지 순차적으로 정보를 전달합니다.

#### 2. 시점별 예측
각 시점 $t$ 에서 모델은 이전에 입력된 단어들의 정보를 바탕으로 바로 다음에 올 단어 $x_{t+1}$ 가 무엇일지 예측합니다.

#### 3. 단계별 손실 계산
매 시점마다 모델이 출력한 예측 분포($\hat{y}^{(t)}$)와 실제 정답 단어($y^{(t)}$)를 비교합니다. 이때 교차 엔트로피 (Cross-Entropy) 를 사용하여 모델이 정답을 얼마나 정확하게 맞혔는지 수치화합니다.

#### 4. 전체 평균 손실 산출
문장의 시작부터 끝($T$)까지 발생한 모든 시점의 손실($J^{(t)}$)을 합산한 후, 시점의 수 $T$ 로 나누어 전체 평균 손실을 구합니다.

#### 5. 파라미터 최적화
산출된 평균 손실값을 최소화하는 방향으로 모델의 내부 파라미터인 $\theta$ ($W_h, W_e, U$ 등)를 최적화하여 학습을 완료합니다.

### 🟩 손실 함수
RNN의 손실 함수는 모델의 예측값과 실제값 사이의 차이를 측정하는 핵심 지표입니다.

#### ◼️ 시점 $t$ 에서의 손실 ($J^{(t)}(\theta)$)
각 단계에서 발생하는 손실은 다음과 같은 교차 엔트로피 공식을 사용합니다.

$$J^{(t)}(\theta) = CE(y^{(t)}, \hat{y}^{(t)}) = -\sum_{w \in V} y_w^{(t)} \log \hat{y}_w^{(t)}$$
   
* $y^{(t)}$ **(정답)**: 실제 다음 단어를 의미하며, 해당 단어의 위치만 $1$ 이고 나머지는 $0$ 인 **원-핫 벡터(One-hot vector)** 형태입니다.

* $\hat{y}^{(t)}$ **(예측)**: 모델이 소프트맥스 함수를 통해 출력한 단어별 확률 분포입니다.

정답 벡터 $y^{(t)}$ 가 원-핫 벡터이기 때문에, 위 식은 실제 정답 단어 $x_{t+1}$ 에 해당하는 로그 예측 확률값에 마이너스를 붙인 $-\log \hat{y}_{x_{t+1}}^{(t)}$ 로 단순화됩니다. 즉, 모델이 정답 단어에 대해 높은 확률을 부여할수록 손실값은 0에 가까워집니다.

#### ◼️ 전체 평균 손실 ($J(\theta)$)
학습의 최종 목표는 특정 시점이 아닌, 코퍼스 전체에 대한 오차를 줄이는 것입니다. 이를 위해 모든 시점의 손실을 평균냅니다.

$$J(\theta) = \frac{1}{T} \sum_{t=1}^{T} J^{(t)}(\theta) = \frac{1}{T} \sum_{t=1}^{T} -\log \hat{y}_{x_{t+1}}^{(t)}$$

이 전체 평균 손실이 최소가 되도록 경사 하강법 등을 적용하여 가중치를 갱신함으로써, 모델은 문맥에 맞는 다음 단어를 더 정확하게 예측할 수 있게 됩니다.

### 🟩 Teacher forcing
RNN은 구조적으로 이전 시점의 출력이 다음 시점의 입력으로 연결되는 순차적 구조를 가집니다. 만약 학습 초기 단계에서 모델이 잘못된 단어를 예측할 경우, 이 오답이 다음 시점의 입력으로 들어가면서 전체 문장의 맥락이 어긋나는 **오차 누적 (Error Accumulation)** 현상이 발생합니다.

* **개념:** 모델이 어떤 오답을 내놓든 상관없이, 다음 시점의 입력으로 실제 정답 (Ground Truth) 을 강제로 주입하는 기법입니다.
* **효과:** 학습 초기에 모델이 길을 잃지 않도록 잡아주는 '보조 바퀴' 역할을 하여 학습 속도를 비약적으로 높여줍니다.
* **주의사항:** 학습 시에는 정답을 보고 배우지만 실제 추론 단계에서는 정답이 없으므로, 학습과 추론 사이의 괴리가 발생하는 노출 편향 (Exposure Bias) 문제가 발생할 수 있습니다.

### 🟩 SGD를 사용한 기울기 계산
이론적으로 모든 데이터를 한꺼번에 계산하느 배치 경사 하강법 (Batch Gradient Descent) 이 정확할 것 같지만, 대규모 코퍼스를 다루는 현대 딥러닝에서는 비효율적입니다.

* **문제점:** 수억 개의 단어를 가진 코퍼스 전체의 기울기를 한 번에 계산하려면 막대한 메모리와 시간이 소요됩니다.
* **해결책 (SGD):** 전체 데이터 중 무작위로 추출한 소량의 샘플(미니 배치)에 대해서만 기울기를 계산하고 가중치를 즉시 업데이트합니다.
* **장점:** 훨씬 적은 메모리로도 학습이 가능하며, 가중치 업데이트가 빈번하게 일어나므로 학습 곡선이 요동치더라도 결과적으로는 더 빠르게 최적점에 도달할 수 있습니다.

### 🟩 BPTT (Backpropagation Through Time)
RNN 의 가중치 $W_h$ 는 모든 시점에서 공유(Shared)됩니다. 따라서 특정 시점의 오차를 줄이기 위해서는 해당 가중치가 과거의 모든 시점에 어떤 영향을 끼쳤는지 추적해야 합니다.

* **기울기 합산:** 시점 $t$ 에서의 손실 $J^{(t)}$ 에 대한 가중치 $W_h$ 의 기울기는, 그 시점까지의 모든 기여도를 합산하여 구합니다.
* **공식:**

$$\frac{\partial J^{(t)}}{\partial W_h} = \sum_{i=1}^{t} \frac{\partial J^{(t)}}{\partial W_h} \bigg|_{(i)}$$

* **작동 원리:** **다변수 연쇄 법칙 (Multivariable Chain Rule)** 에 따라, 현재의 오차는 시간을 거슬러 올라가며(Backpropagate) 각 시점의 가중치에 기울기를 전달합니다. 이를 통해 모델은 현재의 정답을 맞히기 위해 과거의 정보를 어떻게 처리해야 할지 학습하게 됩니다.

## 🚀 RNN 을 이용한 텍스트 생성
학습이 완료된 RNN 언어 모델은 특정 시점까지의 문맥을 바탕으로 다음 단어의 조건부 확률 분포를 계한할 수 있습니다. 이를 반복적으로 수행하여 새로운 시퀀스를 만들어내는 과정을 **롤 아웃 (Generating roll outs)** 이라고 하며, 다음과 같은 4단계 프로세스로 진행됩니다.

### 🟩 텍스트 생성의 4단계 프로세스 

#### 1. 시작 (Initialization)

문장의 시작을 알리는 특수 토큰인 \<s\> (또는 \<START\>)를 모델의 첫 번째 입력($x^{(1)}$)으로 주입합니다. 아 단계에서 모델의 은닉 상태 (Hidden State)는 대개 0 또는 학습된 초기 벡터로 설정되어 생성의 기초가 됩니다.

#### 2. 출력 및 샘플링 (Output & Sampling)

모델은 소프트맥스(Softmax) 함수를 통해 어휘 사전 내의 모든 단어에 대한 확률 분포 $\hat{y}^{(t)}$ 를 출력합니다.

* **그리디 서치 (Greedy Search):** 단순히 확률이 가장 높은 단어 하나만 선택하는 방식입니다. 결과가 고정적이고 단조로울 수 있습니다.
* **확률적 샘플링 (Stochastic Sampling):** 확률 분포에 따라 단어를 무작위로 추출합니다. 낮은 확률의 단어도 선택될 가능성을 열어두어, 문장의 다양성과 창의성을 높이는 데 필수적입니다.

#### 3. 자기회귀적 피드백 (Autoregressive Feedback Loop)

시점 $t$ 에서 샘플링된 단어를 다음 시점 $t+1$ 의 입력으로 다시 주입합니다. 이를 통해 모델은 자신이 바로 이전에 생성한 단어를 문맥 정보로 활용하며, 논리적 흐름을 유지한 채 다음 단어를 예측하게 됩니다.

#### 4. 종료 (Termination) 

모델이 문장의 끝을 알리는 특수 토큰인 \</s\> (또는 \<END\>)를 생성하거나, 사전에 설정한 최대 문장 길이에 도달할 때까지 위 과정을 반복합니다. 종료 토큰이 나오면 시퀀스 생성이 완료됩니다.

## 💯 언어 모델의 평가 지표
언어 모델의 성능을 정량적으로 측정하는 가장 대표적인 표준 지표는 Perplexity (PPL) 입니다. 단어 뜻 그대로 "헷갈리는 정도"를 의미하며, 모델이 다음 단어를 예측할 때, 얼마나 확신을 가지고 있는지를 나타냅니다.

### 🟩 Perplexity의 정의와 수학적 의미
PPL은 문장 내 각 단어에 대한 조건부 확률의 기하평균에 역수를 취한 값으로 정의됩니다. PPL이 낮을수록 모델이 다음 단어를 더 확실하고 정확하게 예측한다는 의미이므로 좋은 모델로 평가받습니다.

$$PPL = \prod_{t=1}^{T} \left( \frac{1}{P_{LM}(x^{(t+1)} | x^{(t)}, \dots, x^{(1)})} \right)^{1/T}$$

* **$P_{LM}$ (예측 확률):** 모델이 실제 정답 단어에 부여한 확률입니다. 이 확률이 높을수록 분모가 커져 전체 PPL 값은 작아집니다.
* **역수 ($1/P$):** 모델이 정답을 맞출 확률이 낮을수록 지표가 커지도록 설계된 장치입니다.
* **기하평균 ($1/T$):** 문장이 길어질수록 확률의 누적 곱은 기하급수적으로 작아집니다. 이를 단어 수 $T$로 정규화하여, 문장 길이에 상관없이 모델의 성능을 객관적으로 비교할 수 있게 합니다.


### 🟩 손실 함수($J$)와의 상관관계
실제 딥러닝 구현 및 학습 과정에서 PPL은 우리가 최소화하고자 하는 **교차 엔트로피 손실 함수(Cross-Entropy Loss)** 과 직결됩니다.

$$PPL = \exp(J(\theta)) = \exp \left( \frac{1}{T} \sum_{t=1}^{T} - \log \hat{y}_{x_{t+1}}^{(t)} \right)$$

* **지수 함수 관계:** PPL은 손실 함수 $J$ 의 지수 ( $e^J$ ) 형태입니다. 즉, 학습 과정에서 손실값 (Loss) 을 줄이는 행위는 수학적으로 PPL 을 개선하는 것과 완벽히 동일한 목표를 가집니다.
 
* **직관적 해석:** $Loss$ 가 0에 가까워질수록 PPL 은 이론적 최소값인 1에 수렴합니다. 이는 모델이 단 한 번의 망설임 없이 매 순간 완벽하게 정답을 맞히는 이상적인 상태를 의미합니다.
