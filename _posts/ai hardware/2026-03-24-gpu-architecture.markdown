## Introduction
우리는 일반적으로 프로그램이 순차적으로 실행된다고 생각한다. 프로세서는 명령어를 하나씩 가져와(fetch), 해석하고(decode), 실행(execute)하는 과정을 반복하며 작업을 수행한다. 이러한 방식은 단일 작업을 빠르게 처리하는 데에는 효과적이지만, 동시에 많은 데이터를 처리해야 하는 상황에서는 한계를 드러낸다.

이러한 한계를 극복하기 위해 CPU는 파이프라이닝, 캐시, 프리페칭, Out-of-order execution과 같은 다양한 최적화 기법을 발전시켜 왔다. 하지만 이러한 기법들은 본질적으로 단일 스레드의 실행 속도를 높이는 데 초점을 맞추고 있어, 처리해야 할 데이터의 양이 폭발적으로 증가하는 환경에서는 충분한 성능 향상을 제공하기 어렵다.

특히 딥러닝은 수많은 연산을 동시에 처리하는 데이터 **병렬성**이 핵심이기 때문에, 기존의 CPU 중심 구조로는 효율적인 처리가 어렵다. 이러한 배경에서 등장한 것이 바로 GPU다. GPU는 하나의 작업을 빠르게 처리하기보다는, 많은 작업을 동시에 처리하는 데 최적화된 구조를 가지고 있다.

그래서 이번 글에서는 CPU와 GPU의 설계 방식을 비교하며, GPU가 어떻게 대규모 병렬 처리를 가능하게 하는지 그 구조와 동작 방식을 정리해보겠다.

## ⚙️ 1. 프로세서와 성능의 기본 개념
GPU 아키텍처를 이해하기 위해서는 먼저 프로세서가 어떻게 동작하고, 성능을 어떤 기준으로 평가하는지에 대한 기본 개념을 정리할 필요가 있다.

### 1.1 Program과 Processor
프로그램은 명령어(instruction)의 연속으로 이루어져 있으며, 프로세서는 이를 순차적으로 실행한다. 일반적인 실행 과정은 다음과 같다.

1. **Fetch :** 메모리에서 다음에 실행할 명령어를 가져온다.

2. **Decode :** 명령어를 해석하여 어떤 연산을 수행할지 결정한다.

3. **Execute :** 실제 연산을 수행한다.

4. **Write Back:** 결과를 레지스터나 메모리에 저장한다.

이러한 흐름은 모든 프로세서의 기본적인 실행 모델이며, CPU와 GPU 모두 이 구조를 기반으로 동작한다.

### 1.2 Latency vs Throughput
프로세서 성능을 이해할 때 가장 중요한 두 가지 지표는 **Latency**와 **Throughput**이다.

#### ◼️ Latency (지연 시간)
* **개념 :** 하나의 작업을 완료하는 데 걸리는 시간이다.
* **목표 :** 개별 작업을 가능한 한 빠르게 끝내는 것을 목표로 한다.

#### ◼️ Throughput (처리량)
* **개념 :** 단위 시간당 처리할 수 있는 작업의 양이다.
* **목표 :** 동시에 많은 작업을 처리하는 것을 목표로 한다.

일반적으로 CPU는 Latency를 최소화하는 방향으로 설계되고, GPU는 Throughput을 극대화하는 방향으로 설계된다.

## 💾 2. CPU Architecture의 한계
GPU 아키텍처를 이해하기 위해서는 먼저 CPU가 어떤 방식으로 성능을 향상시켜 왔는지, 그리고 그 접근 방식이 어떤 한계를 가지는지를 살펴볼 필요가 있다.

### 2.1 CPU의 설계 방식
CPU는 기본적으로 **Latency를 최소화하는 방향**으로 설계된다. 즉, 하나의 작업을 가능한 한 빠르게 처리하는 것이 핵심 목표다.

이를 위해 CPU는 단일 스레드의 실행 속도를 극대화하는 방식으로 발전되어 왔다.

### 2.2 CPU 성능 향상 기법
CPU는 단일 스레드 성능을 극대화하기 위해 다음과 같은 방식으로 성능을 개선해왔다.

#### ◼️ Pipelining
<img src="/images/AIhardware/gpu_arc_1.jpg" style="width:70%;">
* **특징 :** 명령어 실행 과정을 여러 단계로 나누어 겹쳐서 실행하는 방식이다.
* **장점 :** 여러 명령어를 동시에 처리할 수 있어 전체 처리량(throughput)을 증가시킨다.
* **단점 :** Latency 자체를 줄여주지는 않으며, 데이터 의존성이 존재할 경우 pipeline stall이 발생하여 성능 향상이 제한된다.

#### ◼️ Cache
<img src="/images/AIhardware/gpu_arc_2.jpg" style="width:70%;">
* **특징 :** 자주 사용하는 데이터를 속도가 빠른 메모리에 저장하는 구조이다.
* **장점 :** 메모리 접근 Latency를 줄여 전체 실행 속도를 향상시킨다.
* **단점 :** Cache miss 발생 시 오히려 큰 지연이 발생하며, Locality가 낮은 경우 효과가 제한된다.

#### ◼️ Prefetching
<img src="/images/AIhardware/gpu_arc_3.jpg" style="width:70%;">
* **특징 :** 앞으로 사용할 데이터를 미리 예측하여 메모리에서 가져오는 방식이다.
* **장점 :** 메모리 접근 지연을 숨겨 실행 흐름을 끊기지 않게 유지한다.
* **단점 :** 잘못된 예측 시 불필요한 메모리 접근이 발생하여 오히려 성능이 저하될 수 있다.

#### ◼️ Out-of-order Execution & Superscalar
<img src="/images/AIhardware/gpu_arc_4.jpg" style="width:70%;">
* **특징 :** 독립적인 명령어를 순서를 바꿔 병렬로 실행하는 방식이다.
* **장점 :** Instruction-Level Parallelism(ILP)을 활용하여 단일 스레드 성능을 극대화한다.
* **단점 :** 데이터 의존성이 강한 코드에서는 병렬 실행 가능한 명령어가 부족하여 효과가 제한된다.

### 2.3 CPU의 근본적인 한계
앞서 살펴본 최적화 기법들은 모두 단일 스레드의 실행 속도를 높이기 위한 접근이다. 하지만 이러한 방식은 다음과 같은 근본적인 한계를 가진다.

#### ◼️ ILP (Instruction-Level Parallelism)의 한계
* **특징 :** CPU는 하나의 스레드 내에서 독립적인 명령어를 찾아 병렬 실행한다.
* **한계 :** 실제 프로그램은 데이터 의존성이 강하기 때문에 동시에 실행 가능한 명령어 수가 제한된다. 즉, 하드웨어를 아무리 늘려도 활용할 수 있는 병렬성이 부족하게 된다.

#### ◼️ Memory Latency 문제
* **특징 :** DRAM 접근은 여전히 매우 느리다.
* **한계 :** Cache와 Prefetching으로 완화는 가능하지만 완전히 제거할 수는 없다. 따라서 메모리 접근이 병목이 되어 전체 성능을 제한한다.

#### ◼️ 병렬성 확장의 구조적 한계
* **특징 :** CPU는 소수의 강력한 코어로 구성된다.
* **한계 :** 대규모 데이터 병렬 처리를 수행하기에는 구조적으로 불리하다. 따라서 많은 데이터를 동시에 처리하는 작업에는 비효율적이다.

결국 CPU는 단일 작업을 빠르게 처리하는 데에는 매우 강력하지만, 대량의 데이터를 동시에 처리해야 하는 환경에서는 구조적인 한계를 가진다. 이러한 한계를 극복하기 위해 GPU와 같은 병렬 처리 중심 구조가 등장하게 되었다.

## 🚚 3. GPU 아키텍처

### 3.1 GPU 설계 원리
CPU가 단일 작업의 실행 시간을 줄이는 데 집중하는 것과 달리, GPU는 전체 처리량(Throughput)을 극대화하는 방향으로 설계된다. 즉, 개별 작업이 다소 느리더라도 많은 작업을 동시에 처리하는 것이 핵심이다.

이를 위해 GPU는 다음과 같은 설계 원리를 따른다.

#### ◼️ Massive Parallelism
* **개념 :** 수천 개 이상의 스레드를 동시에 실행하여 데이터를 병렬로 처리하는 구조다.
* **특징 :** 대량의 데이터를 병렬로 처리하여 전체 처리량을 크게 증가시킨다.

#### ◼️ Throughput-oriented Design
* **개념 :** 단일 작업 속도보다 전체 처리량을 우선하는 설계 방식이다.
* **특징 :** 개별 연산의 Latency보다 지속적인 연산 수행을 통해 전체 성능을 높인다.

#### ◼️ Latency Hiding
* **개념 :** 메모리 접근으로 인한 지연을 다른 스레드 실행으로 숨기는 방식이다.
* **특징 :** 일부 스레드가 대기 상태일 때 다른 스레드를 실행하여 연산 유휴 시간을 최소화한다.

### 3.2 GPU Hardware 구조
GPU는 대규모 병렬 처리를 위해 다수의 연산 유닛을 계층적으로 구성한 구조를 가진다. 전체적으로 GPU는 여러 개의 **Streaming Multiprocessor (SM)** 로 이루어져 있으며, 각 SM은 다수의 연산 코어를 포함한다.
<img src="/images/AIhardware/gpu_arc_5.jpg" style="width:35%;">

#### ◼️ Streaming Multiprocessor (SM)
* **정의 :** GPU에서 스레드를 실행하고 연산을 수행하는 기본 단위다.
* **특징 :** 다수의 CUDA Core, 레지스터 파일, Shared Memory 등을 포함하며, 여러 개의 thread block을 동시에 처리할 수 있다.

#### ◼️ CUDA Core
* **정의 :** GPU에서 산술 연산을 수행하는 기본 연산 유닛이다.
* **특징 :** 단일 스레드의 연산을 처리하는 스칼라 프로세서이며, 수많은 코어가 병렬로 동작하여 높은 처리량을 제공한다.

#### ◼️ Tensor Core
* **정의 :** 행렬 곱셈과 같은 dense 연산을 가속하기 위해 설계된 특수 연산 유닛이다.
* **특징 :** 딥러닝에서 핵심적인 연산인 Matrix Multiply를 효율적으로 수행하며, Mixed Precision 연산을 통해 성능을 크게 향상시킨다.

#### ◼️ Memory
* **정의 :** GPU는 다양한 속도와 용도를 가진 메모리 계층을 포함한다.
* **특징 :** Register, Shared Memory, Global Memory 등으로 구성되며, 데이터 접근 패턴에 따라 성능이 크게 달라진다.

### 3.3 GPU Software 구조
GPU는 수많은 스레드를 효율적으로 관리하고 실행하기 위해 계층적인 프로그래밍 모델을 사용한다. 이 모델은 **Thread → Block → Grid**로 이어지는 구조를 가지며, 이를 통해 대규모 병렬 처리를 구현한다.
<img src="/images/AIhardware/gpu_arc_6.jpg" style="width:35%;">

#### ◼️ Kernel
* **정의 :** GPU에서 실행되는 함수로, CPU에서 호출되어 병렬로 실행된다.
* **특징 :** 하나의 Kernel 호출은 다수의 Thread를 생성하며, 동일한 연산을 여러 데이터에 동시에 적용한다.

#### ◼️ Thread
* **정의 :** GPU에서 실행되는 가장 작은 단위의 실행 흐름이다.
* **특징 :** 각 Thread는 서로 독립적으로 동작하며, 하나의 데이터 요소를 처리하는 역할을 가진다.

#### ◼️ Thread Block
* **정의 :** 함께 실행되는 thread들의 집합이다.
* **특징 :** 같은 block 내의 Thread들은 Shared Memory를 통해 데이터를 공유할 수 있으며, Synchronization이 가능하다.

#### ◼️ Grid
* **정의 :** 하나의 Kernel 실행에 의해 생성되는 thread block들의 집합이다.
* **특징 :** 각 block은 독립적으로 실행되며, block 간 직접적인 통신은 불가능하고 Global Memory를 통해 간접적으로 데이터 교환한다.

#### ◼️ Warp (SIMT 실행 단위)
* **정의 :** GPU에서 실제로 동시에 실행되는 Thread들의 그룹이며, Warp Scheduler에 의해 관리된다. 일반적으로 32개의 Thread로 구성된다.
* **특징 :** 하나의 Warp 내 Thread들은 동일한 명령어를 동시에 실행하며, 이를 통해 SIMD와 유사한 방식으로 병렬 처리를 수행한다.

## Conclusion
이번에는 GPU 아키텍처를 이해하기 위해, 먼저 프로세서의 기본 개념과 CPU의 설계 방식 및 한계를 살펴보았다. CPU는 단일 작업의 실행 시간을 줄이기 위해 다양한 최적화 기법을 발전시켜 왔지만, Instruction-Level Parallelism의 한계와 메모리 지연 문제로 인해 대규모 병렬 처리에는 구조적인 한계를 가진다.

이러한 한계를 극복하기 위해 등장한 GPU는 Throughput 중심의 설계를 바탕으로, 수많은 스레드를 동시에 실행하는 Massive Parallelism 구조를 채택하고 있다. 또한 SIMT 실행 모델과 Warp Scheduler를 통해 메모리 지연을 숨기고, 연산 유닛을 지속적으로 활용하는 방식으로 높은 성능을 보여준다.

이제 GPU의 구조를 이해했으므로, 다음 글에서는 성능에 직접적인 영향을 미치는 **GPU 메모리 구조**에 대해 자세히 정리해보겠다.
