## ⚙️ Processor & Programs
본격적으로 하드웨어 구조를 알아가기 전에, 그 근간이 되는 프로세서와 프로그램의 관계를 먼저 정리해보도록 하겠다. 프로그램이 실제 물리적인 하드웨어 위에서 어떻게 동작하는지 그 기본 원리를 알아보자.

### 🟩 Processor 란?
**프로세서 (Processor)** 는 명령어를 기반으로 다양한 작업을 수행하는 하드웨어 모듈이며, **Program Counter (PC)** 가 가리키는 메모리 주소를 참조하여 실행해야 할 명령어를 순차적으로 가져와 처리한다. 명령어가 처리될 때마다 레지스터, 메모리, 그리고 CPU 상태 같은 기계 상태 (Machine State)가 실시간으로 변경되며 프로그램이 동작하게 된다.

### 🟩 명령어 실행 단계 (Processor Pipeline)
하나의 명령어가 완전히 처리되기 위해서는 보통 4가지 단계를 거친다. 이것을 **프로세서 파이프라인 (Processor Pipeline)** 이라고 하며, 기본적으로 **Sequential (순차적)** 하게 동작하는 것이 원칙이다. 

1. **Instruction Fetch (IF):** 메모리로부터 실행할 명령어를 프로세서로 가져오는 단계

2. **Instruction Decode (ID):** 가져온 명령어가 어떤 동작을 의미하는지 해석하는 단계

3. **Instruction Execute (IE):** 해석된 내용을 바탕으로 명령어를 처리하는 수행하는 단계

4. **Write-back Result (WB):** 연산이 완료된 결과값을 레지스터나 메모리에 다시 저장하는 단계

### 🟩 하드웨어 성능 지표
하드웨어의 성능을 평가할 때는 크게 두 가지 지표를 사용한다.

#### ◼️ Latency (지연 시간)
**Latency** 는 단일 작업을 완료하는 데 걸리는 순수한 시간, 즉 '빠르기'에 초점을 맞춘 지표다. 입력 ($input$)이 들어간 시점부터 최종 출력 ($output$)이 나올 때까지의 시간을 측정하여 구할 수 있다. 대표적으로 **CPU** 가 이 Latency를 최소화하여 개별 작업을 최대한 빠르게 처리하도록 설계된 프로세서이다.

#### ◼️ Throughput (처리량)
**Throughput** 은 단위 시간당 얼마나 많은 작업을 완료할 수 있는지, 즉 '양'에 초점을 맞춘 지표다. 개별 작업이 얼마나 걸리느냐보다, 정해진 시간 내에 처리할 수 있는 총 작업 수가 중요한 것이다. 수많은 데이터를 동시에 처리해야 하는 딥러닝 환경에서는 이 Throughput을 극대화하는 것이 핵심이며, 이것이 바로 **GPU** 가 탄생한 배경이기도 하다.

### 🟩 CPU 성능 최적화 기법 (Latency-optimized)
위에서도 말했듯이 CPU는 단일 스레드 성능에서 뛰어나도록 설계된 하드웨어다. 하나의 명령어를 빠르게 처리하기 위해 네가지 핵심 기법들이 사용되었는데 아래에서 하나씩 알아보자. 이 개념들은 나중에도 사용되니까 알아두자...ㅇ..

#### ◼️ Pipelining (파이프라인)
명령어 실행 단계를 겹쳐서 실행함으로써 프로세서 내부의 유닛들이 쉬지 않고 일하게 만드는 기술이다. 특정 시점에 여러 명령어가 각기 다른 단계에서 동시에 처리되므로 전체적인 처리량(Throughput)을 늘려준다는 장점이 있다. 하지만 Latency 자체를 줄여주지는 않는다.

#### ◼️ Caches (캐시)
데이터의 **Temporal Locality (시간적 지역성)** 를 활용하여 메모리 접근 속도를 개선하는 방식이다. 일반적으로 데이터는 프로세서 외부의 **Off-chip Memory** 에 저장되며, 외부에서 프로세서로 데이터를 불러오는 것은 매우 큰 Latency 를 발생시킨다. 따라서 프로세서 내부에 용량은 작지만 매우 빠른 **On-chip Memory** (Register File, L1/L2/L3 Cache) 를 넣어, 자주 사용하는 데이터를 이 영역에 머물게 하여 메모리 병목 현상을 해결하였다.

#### ◼️ Prefetching (프리페칭)
데이터의 **Spatial Locality (공간적 지역성)** 를 활용하여, 프로세서가 필요로 할 데이터를 미리 예측해 가져오는 기술이다. for문과 같이 규칙적인 데이터 접근 패턴이 보일 경우, 프로세서는 다음에 읽을 데이터를 예측 패턴 (Predict Pattern)을 통해 미리 파악한다. 이를 통해 실제 명령어가 실행되기 전에 데이터를 Off-chip으로부터 미리 가져와(Prefetch) 캐시에 채워둠으로써 대기 시간을 줄일 수 있다.

#### ◼️ Out-of-order Execution & Superscalars
명령어를 단순히 코드 순서대로 실행하는 것이 아니라, 데이터 의존성(Data dependency)이 없는 독립적인 명령어(Independent instruction)들을 동적으로 찾아내어 병렬적으로 실행하는 방식이다.
