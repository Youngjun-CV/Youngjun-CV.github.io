## 🤓 사용자의 '미래'를 예측하는 별점 예측 시스템
영화 추천 사이트에 들어가 보면 수많은 사용자가 남긴 평점 기록을 볼 수 있습니다. 이 평점들을 분석하면 우리는 사용자의 취향을 파악할 수 있죠. 그렇다면 사용자가 아직 보지 않은 영화에 대해서도 "이 영화를 보면 몇 점을 줄 것이다"라고 미리 예측해 볼 수는 없을까요?

이것을 가능하게 하는 것이 바로 **별점 예측 시스템 (Rating Prediction System)** 입니다. 이번에는 유사도 기반의 별점 예측 방법부터 잠재 요인 모델까지, 시스템이 어떻게 우리의 취향을 수치화하는지 알아보겠습니다.

---

## 🔎 유사도 기반 별점 예측 (Similarity-based)
유사도 지표를 직접적으로 활용하는 가장 직관적인 방법으로, 사용자가 과거에 평가한 아이템들과 예측하려는 아이템 사이의 유사도를 이용합니다.

### (1) 예측값 계산 원리와 예시
사용자 $u$ 가 아이템 $i$ 에 매길 별점은 그가 기존에 평가한 다른 아이템들의 별점을 **유사도 가중 평균** 하여 구합니다.

$$r_{xi} = \frac{\sum_{j \in N} s_{ij} \times r_{xj}}{\sum_{j \in N} |s_{ij}|}$$

* **공식 설명:** 분자는 유사도의 가중합을, 분모는 정규화(Normalization)를 담당합니다.
* **적용 예시:**
  * **예측 내용:** 사용자 $x$ 가 영화 $a$에 줄 점수를 예측해 봅시다. ($N = 2$)
  * **유사도 (영화 $a$ 기준):** 영화 $b$와의 유사도: $0.47$, 영화 $c$와의 유사도: $0.4$
  * **사용자 x의 평점:** 영화 $b$의 평점: $4$점, 영화 $c$의 평점: $5$점
  * **예측 결과:** $\frac{(0.47 \times 4) + (0.4 \times 5)}{0.47 + 0.4} = \frac{1.88 + 2.0}{0.87} \approx$ **4.46** 점

### (2) 다양한 휴리스틱 적용 가능
유사도 기반 별점 예측에는 다양한 휴리스틱을 적용할 수 있습니다. 하지만 일반적으로 아래의 방법들이 많이 사용됩니다.
* **아이템 기반 협력 필터링:** **사용자 유사도 (User-User)** 보다 연산 효율성이 높고 아이템 상태가 상대적으로 안정적인 **아이템 유사도 (Item-Item Collaborative Filtering)** 방식을 주로 사용합니다.
* **가중치 산정:** 유사도 지표 중 유저의 평점 편향을 잘 보정해 주는 **피어슨 상관계수 (Pearson Corr.)** 가 가중치 계산에 널리 쓰입니다.

## 📊 모델 기반 별점 예측 (Model-based)
**모델 기반 별점 예측**은 데이터 속에 숨겨진 사용자와 아이템의 패턴을 수학적 함수 $f(u,i)$ 로 모델링하여 아직 평가되지 않은 별점을 예측하는 방식입니다. 유사도 기반 방식과 달리 데이터를 사전에 학습하기 때문에 대규모 데이터셋에서도 연산 효율이 뛰어나며 더욱 정교한 개인화가 가능합니다.

### (1) 가장 단순한 모델 (Global Baseline)
* **모델:**
$f(u, i) = \alpha$
* **설명:** 모든 사용자와 아이템에 대해 단 하나의 값인 **전체 평균 별점** 으로 예측하는 가장 기초적인 모델입니다.

### (2) 사용자와 아이템의 성향 (Bias) 모델
사용자가 평소 점수를 후하게 주는지, 특정 아이템이 원래 평점이 높은지 등 개별 성향을 반영합니다.

$$f(u, i) = \alpha + \beta_u + \beta_i$$

* $\beta_u$: 사용자 $u$ 의 성향 (평균에 비해 얼마나 별점을 잘 주는가?)
* $\beta_i$: 아이템 $i$ 의 성향 (평균에 비해 얼마나 별점을 높게 받는가?)

### (3) 정규화 (Regulation)와 최적화
예측 오차 (Error) 를 최소화하되, 모델 파라미터 값이 너무 커져 발생하는 **과적합 (Overfitting)** 을 막기 위해 **정규화 항** 을 추가합니다.

$$arg \min_{\alpha, \beta} \sum_{u, i} (\alpha + \beta_u + \beta_i - R_{u,i})^2 + \lambda [ \sum_u \beta_u^2 + \sum_i \beta_i^2 ]$$

이 모델에서 **기울기가 0이 되는 지점**을 찾아 미분하면 다음과 같이 $\beta$ 값을 구할 수 있습니다.

$$\beta_u = \frac{\sum_{i \in I_u} R_{u,i} - (\alpha + \beta_i)}{\lambda + |I_u|}$$

$$\beta_i = \frac{\sum_{i \in U_i} R_{u,i} - (\alpha + \beta_u)}{\lambda + |U_i|}$$

### (4) Bias 모델 최적화 예시
* **설정:** 전체 평균($\alpha$) = 3.0, 정규화 계수($\lambda$) = 2.0
* **데이터:** 사용자 $u$ 는 영화 $1$ (4점, $\beta_1 = 0.5$) 과 영화 $2$ (5점, $\beta_2 = 1.0$) 를 평가함.
* **업데이트 로직:** 사용자 $u$ 가 평가한 별점에서 평균과 아이템 편향을 뺀 '잔차'의 합을 구함.
  * 영화 1 잔차: $4.0 - (3.0 + 0.5) = 0.5$
  * 영화 2 잔차: $5.0 - (3.0 + 1.0) = 1.0$
* **최종 계산:**
$$\beta_u = \frac{0.5 + 1.0}{2.0 \text{ (정규화)} + 2 \text{ (평가 개수)}} = \frac{1.5}{4.0} = \mathbf{0.375}$$
* **의미:** 사용자 $u$ 는 평균보다 약 **0.375점** 정도 점수를 더 후하게 주는 경향이 있다고 모델이 학습함.

## 🚀 잠재 요인 모델 (Latent Factor Model)
사용자와 아이템의 성향을 모델링하는 것은 효과적이지만, 여전히 아이템과 사용자를 **따로따로 모델링** 하고 있다는 한계가 있습니다. 즉, "사용자가 특정 장르의 영화를 선호한다"와 같은 구체적인 **상호작용 (Interaction)** 정보는 담지 못합니다. 이를 해결하기 위해 등장한 것이 **잠재 요인 모델** 입니다.

### (1) 잠재 요인 (Latent Factor) 모델
사용자와 아이템의 속성을 동일한 차원의 벡터로 표현하여 둘 사이의 특수한 선호 관계를 모델링합니다.

$$f(u, i) = \alpha + \beta_u + \beta_i + \gamma_u \cdot \gamma_i$$

* **$\gamma_u, \gamma_i$:** 사용자 및 아이템의 **잠재 벡터 (Latent Vector)**

### (2) 정규화 (Regulation)와 최적화
잠재 요인 모델 역시 Bias 모델처럼 **과적합 방지**를 위해 **L2 Norm 기반의 정규화 항**을 추가하여 모델을 최적화합니다.

$$\arg \min_{\alpha, \beta, \gamma} \sum_{u, i} \left(\alpha + \beta_u + \beta_i + \gamma_u \cdot \gamma_i - R_{u,i}\right)^2 + \lambda \left[ \sum_u \beta_u^2 + \sum_i \beta_i^2 + \sum_u \|\gamma_u\|^2 + \sum_i \|\gamma_i\|^2 \right]$$

하지만 이 모델에서의 최적화 문제는 **Convex (볼록 함수)** 가 아닙니다. 따라서 수학적으로 단 한번에 전역 최적해 (Global Optimum)을 구하는 것이 사실상 불가능하며, 근사치를 구하는 정교한 알고리즘이 필요합니다.

### (3) Alternating Least Squares (ALS)
Latent Factor Model 은 **Convex** 하지 않기 때문에 전역 최적해를 한 번에 구하기 어렵다는 문제가 있었습니다. 이 문제를 해결하기 위해 변수를 번갈아 가며 고정하여 푸는 ALS 기법을 적용합니다.

* **핵심 아이디어:** 사용자 또는 아이템 매트릭스 중 하나를 상수로 고정하면 문제는 **Convex** 가 되어 풀기 쉬워집니다.
* **적용 과정:**
  * **Step 1 (고정):** 현재 사용자들의 잠재 벡터($\gamma_u$)를 **상수(Fixed)** 로 취급합니다.  이제 문제는 아이템 벡터에 대한 단순한 선형 회귀 문제가 됩니다.
  * **Step 2 (계산):** 아이템 $i$ 를 평가한 모든 유저들의 벡터와 별점 데이터를 사용하여, 에러를 최소화하는 최적의 $\gamma_i$ 를 도출합니다.
  * **Step 3 (교대):** 이번에는 방금 구한 $\gamma_i$ 를 고정하고, 사용자 벡터($\gamma_u$)를 순차적으로 업데이트합니다. 이 과정을 반복(Iteration)하면 복잡한 취향의 벡터들이 서로의 관계를 찾아가며 수렴합니다.

## 🫡 마치며: 데이터 너머의 취항을 읽는 법
지금까지 단순 평균에서 시작해 사용자의 고유한 취향을 벡터로 추출하는 **별점 예측 시스템** 에 대해 알아보았습니다. **Latent Factor Model** 은 수학적 최적화 과정이 복잡하고 전역 최적해를 구하기가 까다롭지만, 사용자와 아이템 간의 숨겨진 상호작용 (Interaction) 을 가장 정교하게 포착해낼 수 있는 모델이라는 생각이 듭니다. 감사합니다.
