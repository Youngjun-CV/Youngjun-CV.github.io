## 🙂 들어가며
지난 포스팅에서는 푸르닝 설계의 두 핵심 요소인 **Granularity (푸르닝 단위)** 와 **Criterion (푸르닝 기준)** 에 대해 알아보았다. Granularity가 하드웨어의 연산 효율을 결정하는 구조에 대한 고민이었다면, Criterion은 어떤 가중치가 중요한지를 판단하는 지능에 대한 고민이었다.

하지만 푸르닝을 완성하기 위해 고려해야할 사항이 하나 더 남아 있다. 바로 **Pruning Ratio (푸르닝 비율)** 다. 모델 내의 모든 레이어에 똑같은 비율로 가중치를 쳐내는 것은 결코 효율적이지 않기 때문에, 레이어별로 얼마나 깎을 것인가를 결정하는 것이 푸르닝의 마지막 결정 요소이다.

또한, 아무리 완벽하게 가중치를 0으로 만들었다 하더라도, 이를 받아주는 하드웨어가 0을 효율적으로 처리하지 못한다면 실제 속도 향상은 기대하기 어렵다. 결국 진정한 의미의 모델 경량화는 시스템 레벨에서 지원을 해줘야 비로소 완성된다.

그래서 이번에는 레이어별 최적의 비율을 찾는 방법에 더해서, 하드웨어가 비어있는 공간을 어떻게 실제 연산 가속으로 연결하는지에 대해 하나씩 알아보겠다.

## 📊 Pruning Ratio (푸르닝 비율)
푸르닝의 **단위(Granularity)** 와 **기준(Criterion)** 을 정했다면, 이제 "각 레이어마다 얼마나 깎을 것인가"에 대한 전략, 즉 푸르닝 비율을 고민해야 한다.

### 🟩 Pruning Ratio의 개념
![image](/images/ModelCompression/ratio_1.jpg)
푸르닝 비율이란 모델 내의 각 레이어마다 푸르닝 강도를 얼마나 적용할지 선택하는 것을 말한다. 모든 레이어를 동일한 비율로 줄이는 **Uniform Shrinking** 방식보다, 레이어별 특성에 맞춰 비율을 다르게 설정하는 **Non-uniform Pruning** 방식이 모델의 성능을 훨씬 잘 보존한다는 것이 입증되었다. 즉, 모델 전체의 **Sparsity** 목표가 같더라도 레이어별로 이 비율을 어떻게 배분하느냐에 따라 경량화의 성공 여부가 달라진다.

### 🟩 Finding Pruning Ratio: Sensitivity Analysis
![image](/images/ModelCompression/ratio_2.jpg)
레이어마다의 적절한 비율을 찾기 위해 가장 먼저 수행하는 것이 **Sensitivity Analysis (민감도 분석)** 다. 각 레이어는 입력 데이터로부터 추출하는 정보의 종류가 다르기 때문에 푸르닝에 대한 **Sensitivity (민감도)** 도 제각각이다.

* **민감도가 높다:** 푸르닝을 조금만 해도 모델 전체의 정확도(Accuracy)가 급격히 떨어진다. 일반적으로 이미지의 기초적인 특징을 추출하는 첫 번째 레이어가 민감도가 높은 경향이 있다.
* **민감도가 낮다 (Redundant):** 가중치를 많이 제거해도 성능 변화가 적다. 이것은 해당 레이어에 불필요하거나 중복된 정보가 많다는 것을 의미한다.

#### ◼️ 분석 방법 (예시: VGG-11)
1. **레이어 선택**

    모델의 각 레이어($L_{0}, L_{1}, \dots$)를 순차적으로 선택한다.

2. **푸르닝 적용**

    해당 레이어의 푸르닝 비율을 $0$ 부터 $0.9$ 까지 $0.1$ 단위로 변화시키며 모델 전체의 정확도를 측정한다. 이때 다른 레이어들은 푸르닝하지 않은 상태로 둔다.

3. **민감도 분석**

    비율 변화에 따른 정확도 하락 곡선을 그려 민감도를 비교한다.

위 그래프 예시를 보면 $L_{0}$ 은 약 $70\%$ 지점부터 정확도가 급락하는 반면, $L_{1}$ 은 $80\%$ 이상 푸르닝해도 정확도가 거의 유지된다. 이는 $L_{1}$ 이 상대적으로 **불필요한 정보(Redundancy)** 를 많이 포함하고 있음을 의미한다.

#### ◼️ 결정 방식과 한계
분석 결과를 바탕으로, 사용자는 사전에 설정한 정확도 임계값($Threshold~T$)을 넘지 않는 선에서 레이어별 최대 푸르닝 비율을 결정하게 된다.

하지만 이 방식은 각 레이어를 독립적으로 분석한다는 치명적인 한계가 있다. 실제 환경에서는 여러 레이어가 동시에 푸르닝될 때 발생하는 **상호작용(Joint Sensitivity)** 을 반영하지 못하므로, 사람이 휴리스틱하게 최적의 조합을 찾는 데는 엄청난 시행착오와 비용이 따르게 된다.

### 🟩 AMC: AutoML for Model Compression
<img src="/images/ModelCompression/ratio_3.jpg" style="width:70%;">
이러한 수동 분석의 한계를 극복하고 최적의 비율 조합을 자동화하기 위해 등장한 것이 바로 **AMC** 다. 강화학습 에이전트가 모델의 환경을 탐색하며 최적의 경량화 전략을 스스로 학습한다.

#### ◼️ 작동 메커니즘 (DDPG Agent 기반)
AMC는 모델의 레이어를 하나씩 순차적으로 훑으며 각 레이어에 최적화된 결정을 내린다.

* **State ($s_{t}$):** 현재 결정 중인 레이어의 특징 정보를 담고 있다. 레이어 인덱스($t$), 채널 수($c$), 커널 사이즈($n$), 피처 맵의 크기($h, w$) 등이 포함된다.

* **Action ($a_{t}$):** 에이전트가 내리는 결정으로, 해당 레이어의 **푸르닝 비율 (Pruning Ratio)** 을 의미한다. **DDPG** 에이전트를 사용해서 이산적인 값이 아닌 연속적인 비율 값을 정교하게 출력한다.

* **Reward ($r$):** 모든 레이어의 설정을 마친 후, 푸르닝된 모델을 평가하여 에이전트에게 보상을 준다. 기본적으로 $Reward = -Error$ 를 사용하여, 정확도 하락이 적을수록 큰 보상을 주는 식으로 학습한다. 만약 연산량(FLOPs)이나 지연 시간(Latency)을 제약 조건으로 넣고 싶을 때는 $Reward = -Error \times \log(FLOPs)$ 와 같이 수식을 보정하여, 정확도 보존과 연산량 감소를 동시에 달성하도록 유도하기도 한다.


#### ◼️ AMC의 효과
![image](/images/ModelCompression/ratio_4.jpg)
AMC를 사용하면 사람이 설계한 것보다 더 높은 압축률에서도 정확도를 잘 유지한다. 특히 AMC가 학습한 결과를 분석해 보면, 어떤 레이어가 실제로 모델 성능에 핵심적인 역할을 하는지 시각적으로 확인할 수 있어 모델의 내부 구조를 이해하는 데도 큰 도움을 준다.

## 💻 System Support for Sparsity
푸르닝을 통해 **Sparsity** 를 구현했다고 하더라도, 하드웨어가 이를 적절히 처리하지 못하면 연산 속도 향상을 기대하기 어렵다. 일반적인 프로세서는 0을 만나도 똑같이 곱셈과 덧셈 연산을 수행하기 때문이다. 따라서 시스템 가속을 위해서는 하드웨어가 0을 '건너뛰거나' 압축된 데이터만 '골라 계산'하는 시스템적 지원이 반드시 필요하다.

### 🟩 NVIDIA M:N Sparsity (2:4 Structured Sparsity)
지난 포스팅에서 다뤘던 **Pattern-based Pruning** 이 실제 하드웨어에서 구현되는 방식이다. NVIDIA는 정확도 하락을 최소화하면서도 하드웨어 가속 효율을 극대화하기 위해 **2:4 Sparsity** (4개 중 2개는 반드시 0) 방식을 사용한다.

#### ◼️ 1단계: 행렬 압축 (Compression)
<img src="/images/ModelCompression/ratio_5.jpg" style="width:70%;">
원본 가중치 행렬에서 0이 아닌 값(Non-zero values)들만 골라내어 압축한다. 이때 두 가지 데이터가 생성된다.

* **Non-zero Data Values:** 0을 제외한 실제 가중치 값들만 모은 데이터다. 이를 통해 메모리 크기를 절반으로 줄인다.
* **2-bit Metadata (Indices):** 압축된 값들이 원래 4개의 블록 중 몇 번째 위치(0~3)에 있었는지 기록한 인덱스 정보다. 위치당 딱 2비트만 사용하여 오버헤드를 최소화한다.

#### ◼️ 2단계: Tensor Core 매핑 및 연산
![image](/images/ModelCompression/ratio_6.jpg)
압축된 가중치가 입력 데이터와 만날 때 **Tensor Core** 는 인덱스 정보를 활용하여 효율적인 연산을 수행한다.

1. 입력 행렬에서 인덱스가 가리키는 위치의 값들만 가져온다 **(Gather)**.
2. 추출된 입력값들을 압축된 **Non-zero Data** 와 1:1로 매칭하여 곱한다.
3. 결과적으로 불필요한 0 연산이 완전히 제거되어, 이론적으로 최대 2배의 처리 속도 향상을 얻을 수 있다.

### 🟩 Sparse Convolution on Sparse Inputs
가중치뿐만 아니라 입력 데이터까지 Sparse 할 때, 연산량을 획기적으로 줄이는 방식이다.

#### ◼️ 핵심 개념: Submanifold Sparse Convolution
![image](/images/ModelCompression/ratio_7.jpg)
일반적인 Convolution은 연산을 거칠수록 데이터가 주변으로 퍼지며 Sparsity가 금방 사라진다. 이를 방지하기 위해 **Submanifold** 방식은 입력값 중 0이 아닌 위치에서만 출력을 계산한다. 이를 위해 가중치를 고정해 두고 연산하는 방식을 주로 사용한다.

#### ◼️ 연산 과정: Weight-Stationary Computation
![image](/images/ModelCompression/ratio_8.jpg)
**Weight-Stationary Computation** 방식은 가중치를 연산기에 고정시켜 메모리 접근을 최소화하는 하드웨어 최적화 기법이다. 가중치를 매번 새로 불러올 필요가 없어 메모리 대역폭을 아끼고, 입력의 0인 부분은 연산 시작조차 하지 않아 속도를 비약적으로 높인다는 장점이 존재한다.

1. **Weight Grouping & Maps:** 연산 전에 0이 아닌 입력값들이 어떤 가중치와 곱해져야 하는지 미리 그룹화한다.

2. **Input Buffer:** 입력 특징 맵에서 0이 아닌 유효한 값들만 버퍼에 로드한다.

3. **Weight-Stationary:** 특정 가중치를 하드웨어 연산기에 고정(Stationary)시켜 두고, 해당 가중치와 곱해질 입력값들을 순차적으로 처리한다. 가중치를 매번 메모리에서 불러오는 비용을 아낄 수 있다.

4. **Partial Sum Accumulation:** 계산된 값들을 부분합(Partial Sum) 버퍼에 저장하고, 최종적으로 Output Features 로 합산한다.
고정된 가중치와 버퍼에 로드된 유효 입력값들을 연산한 뒤, 그 결과를 부분합(Partial Sum) 버퍼에 쌓아 최종 **Output Features** 를 완성한다.

## ☺️ 마치며
이로써 푸르닝에 관한 핵심적인 내용들을 모두 살펴보았다. 우리는 푸르닝의 기초적인 개념부터 시작해서, 모델을 어떤 단위로 깎을지(Granularity)와 어떤 기준을 세워 제거할지(Criterion)에 대해 알아봤다. 또한 레이어마다 최적의 비율(Ratio)을 찾아내는 전략을 세웠고, 마지막으로 이렇게 만들어진 희소성(Sparsity)을 하드웨어가 어떻게 실제 연산 가속으로 연결하는지(System Support)까지 정리해 보았다.

푸르닝에 대해 알아보면서 모델 경량화가 단순히 파라미터의 개수를 줄이는 것이 아니라, 소프트웨어 알고리즘과 하드웨어 아키텍처가 밀접하게 상호작용하여 최적의 효율을 찾아내는 과정이란 것을 느꼈다.

이제 가중치의 개수를 줄여 모델을 경량화하는 법을 알아봤으니, 다음 포스팅부터는 가중치의 **표현 정밀도** 를 줄여 모델을 효율적으로 만드는 **Quantization (양자화)** 에 대해 알아보겠다.
